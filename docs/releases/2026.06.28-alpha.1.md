Images:

- `ghcr.io/<owner>/retos-web:2026.06.28-alpha.1`

## Highlights

- Shared backend image for API, worker, and migrations through the `backend-runtime`
  Docker target.
- React operational console for document management, ingestion, indexing, jobs, evals,
  audit events, and grounded query results.
- FastAPI backend with persisted domains, sources, documents, versions, artifacts,
  segments, jobs, progress events, and audit journals.
- RabbitMQ-backed Celery worker flow for ingestion, indexing, OCR, and eval jobs.
- Local-first provider posture with Ollama `gemma4` as the default profile and paid
  providers disabled by default.
- Deep Agents harness integration path with deterministic fallback, controlled corpus
  tools, citations, neighboring context, evidence route, evidence audit, contradiction
  audit, bounded multi-hop subquery execution, multi-hop audit, budget accounting, and
  persisted query journals.
- Deterministic smoke and expanded agent multi-hop evals plus opt-in SQuAD, HotpotQA,
  HotpotQA-agent, Natural Questions, NQ-Open adapter, and OCR benchmark paths, with
  React controls for local dataset suites.
- `hotpotqa-agent` calibration maps local HotpotQA supporting facts into deterministic
  agent audit cases through CLI, admin API, rerun, and React controls without provider
  calls.
- Domain-owned dataset eval runs for auditable calibration history, filtered trends,
  scoped reruns, mixed-scope comparison guards, and React `Eval scope` controls.
- Docker release guardrails for topology, backend runtime image ID, OCI metadata, image
  size budgets, release workflow publishing, release notes, and operations documentation.

## Migration Notes

- Apply Alembic migrations with the shared backend image before starting API and worker:

```bash
RETOS_IMAGE_TAG=2026.06.28-alpha.1 docker compose run --rm migrate migrate
```

- This pre-alpha candidate expects an empty or development RetOS database. Backup before
  upgrading any existing environment.
- Search indexes are rebuildable projections. Rebuild indexes after restore or when
  moving storage volumes between machines.

## Compatibility

- Python: 3.14
- Node: 24
- Postgres: 18-bookworm
- RabbitMQ: 4-management
- Ollama: local `gemma4` profile
- Browser: Chromium-family browser for the validated Playwright smoke path

## Security

- License: MIT.
- Containers run as non-root users.
- Development secrets in `.env.example` are placeholders and must be replaced outside
  local development.
- Paid LLM calls remain disabled unless `RETOS_ALLOW_PAID_LLM=true` is set explicitly
  with provider credentials.
- Release images include OCI source/license/revision labels.
- Python and Node dependency advisories are checked with `make dependency-audit`; the
  current runtime pins include patched `orjson` and `Pillow` versions.
- GHCR publishing workflow requests SBOM/provenance attestations, creates Cosign keyless
  signatures, and runs Cosign signature verification for published image digests.

## Known Limitations

- Product maturity is pre-alpha; phases remain in progress in `planning/04-process-tracker.md`.
- Larger real-dataset trend calibration now has a 100-record/30-case to
  200-record/40-case PASS comparison. Broader public slices beyond 200 records and 40
  cases per target still need trend evidence before release promotion.
- Multi-hop planning is deterministic, auditable, and budgeted; broader learned routing
  heuristics remain planned beyond the alpha.
- Eval reruns depend on persisted runnable payloads.

## Rollback

Rollback is image-tag based:

```bash
export RETOS_IMAGE_TAG=<previous-tag>
docker compose up -d migrate
docker compose up -d api worker web
make docker-smoke
```

Restore Postgres and storage backups if a migration or data-shape change must be undone.
Keep failed release logs, image digests, and eval reports with the incident notes.
